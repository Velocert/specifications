# Velocert Specifications

This repository contains technical specifications developed and maintained by
[Velocert](https://velocert.com) for verifiable credentials, electronic signatures,
and related edTech technologies.

## Repository Structure

```bash
velocert/specifications/
├── specifications/             # Published specifications
├── draft/                      # Draft specifications under development
└── reference-implementations/  # Reference implementations
```

### Published Specifications

Final, stable specifications that have undergone review and are ready for implementation.

**Location:** `/specifications/`

### Draft Specifications

Work-in-progress specifications that are under active development and review. These documents may change significantly before publication.

**Location:** `/draft/`

### Reference implementations

Reference implementation of draft and published specifications.

**Location:** `/reference-implementations/`

## Contributing

We welcome feedback and contributions to our specifications. To contribute:

1. **For issues or suggestions:** Open an issue in this repository
2. **For proposed changes:** Submit a pull request with your proposed modifications
3. **For discussion:** Use the issue tracker for technical discussions

### Contribution Guidelines

- All contributions must comply with the repository license [CC BY-SA 4.0](LICENSE)
- Proposed changes to published specifications should be submitted as issues first
- Draft specifications are open to direct pull requests
- Follow the existing document structure and formatting conventions
- Use RFC 2119 normative language (MUST, SHOULD, MAY) appropriately

## Document Lifecycle

Specifications in this repository follow this lifecycle:

1. **Draft** - Initial development, active changes expected
2. **Review** - Stable draft undergoing community review
3. **Published** - Finalized specification moved to `/specifications/`
4. **Updated** - Published specs may receive updates (versioned appropriately)
